# @zipbul/helmet — Functional & Conformance Specification

> Authoritative basis for defect classification. Anything that violates a
> clause below is a **must-fix**. Anything not stated is **out of scope**.
>
> Last revised: 2026-04-27.

---

## 0. Package Scope

@zipbul/helmet emits and validates **HTTP response security headers** for
servers built on the Web Fetch API (Bun `Bun.serve`, Node 18+, Deno,
Cloudflare Workers, Vercel Edge, Hono, Fastify-fetch, Express via adapter).

### In scope
- Outbound response security headers
- Per-header validation against the cited spec
- Server identification header removal (OWASP)
- CSP report parsing (server-side ingestion of `application/csp-report` and
  `application/reports+json`)
- Cryptographic builders strictly required by the headers above (CSP nonce,
  SRI hash)

### Out of scope (by design)
- Request-side CORS, preflight (→ `@zipbul/cors`)
- Rate limiting, DDoS mitigation (→ `@zipbul/rate-limiter`)
- Authentication, OAuth, JWT, sessions
- CSRF tokens (Double-Submit, Synchronizer)
- Input sanitisation / validation
- Cookie attribute management
- TLS configuration
- Structured logging, metrics, OpenTelemetry hooks

## 1. Common Cross-Cutting Requirements

These apply to **every** feature listed in §2.

### 1.1 Validation timing
- All validation runs at `Helmet.create()`. **Runtime** `headers()` MUST NOT
  throw for issues that could be caught at create time.
- Per-request `headers({ nonce })` MAY validate the nonce string only.

### 1.2 Error contract
- Every validation failure produces a `ViolationDetail`:
  ```
  { reason: HelmetErrorReason, path: string, message: string }
  ```
- Multiple violations are aggregated; `Helmet.create()` throws a single
  `HelmetError` whose `violations` array contains every issue.
- `path` follows dotted notation: `optionKey.subKey[index]`.
- `reason` enum values are stable, machine-readable strings; never throw a
  generic `Error` from public API.

### 1.3 Header-injection guard
- Every user-supplied string interpolated into a header value MUST be
  rejected if it contains C0 controls (`%x00–%x1F`), DEL (`%x7F`), or any
  other character that would terminate / corrupt a header field.

### 1.4 Prototype-pollution guard
- Every option whose value is a `Record<string, …>` from user input MUST
  inspect the prototype chain (`Object.getPrototypeOf`) before iterating
  with `Object.entries`.
- Reserved keys (`__proto__`, `constructor`, `prototype`) MUST be rejected.

### 1.5 DoS bounds (constants.ts `LIMITS`)
- Every iterating validator MUST bound user input by the documented limit.
  Exceeding the limit yields `HelmetErrorReason.InputTooLarge`.

### 1.6 Header byte cap
- After compilation, every emitted header value MUST be ≤
  `LIMITS.headerValueBytes` (16 KiB). Violations yield
  `HelmetErrorReason.HeaderValueTooLarge`.

### 1.7 Immutability
- `Helmet.create()` returns a deep-frozen instance.
- `headers()` returns a fresh `Headers` (mutable per Fetch contract).
- `headersRecord()` returns a shallow-frozen plain object.
- `toJSON()` returns a `DeepReadonly` snapshot.

### 1.8 Strictness rule
- Spec-valid input MUST be accepted (no false positives).
- Spec-invalid input MUST be rejected (no false negatives).
- Stricter-than-spec rejections allowed only when documented (e.g.,
  `hstspreload.org` requirements on top of RFC 6797).

## 2. Features

Twelve major buckets. Each lists **(a) Scope, (b) Conformance target,
(c) Security guarantees, (d) Validation strictness, (e) Defaults,
(f) API contract.**

---

### F1 · Content Security

(a) **Scope** — `Content-Security-Policy`, `Content-Security-Policy-Report-Only`,
plus the `trusted-types` / `require-trusted-types-for` directives.

(b) **Conformance** — W3C CSP Level 3 ED (https://www.w3.org/TR/CSP3/),
W3C Trusted Types (https://w3c.github.io/trusted-types/dist/spec/).

(c) **Security guarantees**
- Nonce generated by `Helmet.generateNonce()` is CSPRNG-backed
  (`crypto.getRandomValues`) and **MUST yield ≥ 128 bits of entropy**
  (16 bytes of randomness, base64url-encoded ≥ 22 chars).
- `'nonce-…'` injection is single-pass, function-form `replaceAll` to
  prevent `$&`-style cache-poisoning.
- `report-to` group names are cross-validated against
  `Reporting-Endpoints` — an undeclared name throws (browsers silently
  drop, masking reports).
- `frame-ancestors` source-list restricted to `scheme-source / host-source
  / 'self' / 'none'` per CSP3 §6.4.2 (no nonces, hashes, or other
  keywords).

(d) **Validation strictness**
- All 20 directives enumerated in CSP3 §6 + WebRTC + Fenced Frame
  variants.
- Source-list grammar: per CSP3 §2.3.1 (host, scheme, keyword, nonce,
  hash sources) — including both standard and url-safe base64 alphabets
  per `base64-value = 1*( ALPHA / DIGIT / "+" / "/" / "-" / "_" ) *2( "=" )`.
- Hash-source length per algorithm: SHA-256 = 43 or 44 chars,
  SHA-384 = 64 chars, SHA-512 = 86 or 88 chars (with 0–2 padding chars).
- Sandbox tokens limited to the 14 canonical CSP3 §6.7.4 values.
- Trusted Types: `tt-policy-name` per spec, `tt-keyword` ∈
  {`'allow-duplicates'`, `'none'`, `*`}.
- Deprecated directives explicitly rejected (`block-all-mixed-content`,
  `plugin-types`, `prefetch-src`, `navigate-to`, `require-sri-for`,
  `referrer`, `reflected-xss`).
- Bare keywords (unquoted `self`, `none`) rejected with actionable
  message.
- Per-directive source cap: `LIMITS.cspSourcesPerDirective` (64).
- Total directive cap: `LIMITS.cspDirectiveKeys` (32).

(e) **Defaults** — when `contentSecurityPolicy` is omitted or `true`,
emit OWASP-aligned: `default-src 'self'; base-uri 'self'; form-action 'self';
frame-ancestors 'none'; object-src 'none'; manifest-src 'self';
upgrade-insecure-requests`.

(f) **API contract** — `HelmetOptions.contentSecurityPolicy`,
`HelmetOptions.contentSecurityPolicyReportOnly`. Both accept `false |
ContentSecurityPolicyOptions`. `Helmet.csp(input?)` static helper.

### F2 · Transport Security (HSTS)

(a) **Scope** — `Strict-Transport-Security`.

(b) **Conformance** — RFC 6797 §6.1; hstspreload.org submission rules.

(c) **Security guarantees** — `preload: true` enforces hstspreload.org
requirements (`max-age ≥ 31_536_000` AND `includeSubDomains`) at create
time so a misconfigured preload claim is impossible.

(d) **Validation strictness**
- `max-age` is non-negative integer (delta-seconds per RFC 6797).
  `max-age=0` accepted (means "delete current entry").
- Directive serialised as token (unquoted) in conventional order
  `max-age; includeSubDomains; preload`.

(e) **Defaults** — `max-age=63072000; includeSubDomains` (Mozilla 2-year
recommendation). `preload` is opt-in.

(f) **API contract** — `HelmetOptions.strictTransportSecurity`.
`Helmet.hsts(input?)`.

---

### F3 · Click-jacking Defence

(a) **Scope** — `X-Frame-Options` + CSP `frame-ancestors`.

(b) **Conformance** — RFC 7034 (X-Frame-Options); CSP3 §6.4.2
(frame-ancestors).

(c) **Security guarantees**
- `frame-ancestors` MUST forbid nonces, hashes, and every keyword other
  than `'self'` / `'none'` (per CSP3 §6.4.2 ABNF).
- `X-Frame-Options` `ALLOW-FROM` (deprecated) explicitly rejected.

(d) **Validation strictness** — XFO ∈ {`deny`, `sameorigin`} (case
preserved for WAF compatibility).

(e) **Defaults** — XFO = `deny`; CSP frame-ancestors = `'none'`.

(f) **API contract** — `HelmetOptions.xFrameOptions`,
`HelmetOptions.contentSecurityPolicy.directives.frameAncestors`.
`Helmet.xFrameOptions(input?)`.

---

### F4 · MIME Confusion Defence

(a) **Scope** — `X-Content-Type-Options`.

(b) **Conformance** — WHATWG Fetch (`nosniff` semantics) + MIME Sniffing
spec.

(c) **Security guarantees** — emit only the literal token `nosniff`.

(d) **Validation strictness** — boolean `true`/`false` only.

(e) **Defaults** — `true` (header emitted).

(f) **API contract** — `HelmetOptions.xContentTypeOptions`.
`Helmet.xContentTypeOptions()`.

---

### F5 · Referrer Disclosure Control

(a) **Scope** — `Referrer-Policy`.

(b) **Conformance** — W3C Referrer Policy §4.1, §8.1.

(c) **Security guarantees** — empty token list rejected (W3C §4.1
`1#policy-token` requires ≥ 1 token).

(d) **Validation strictness** — exactly the 8 valid tokens
(`no-referrer`, `no-referrer-when-downgrade`, `origin`,
`origin-when-cross-origin`, `same-origin`, `strict-origin`,
`strict-origin-when-cross-origin`, `unsafe-url`). Multi-token fallback
list joined with `, ` per §8.1.

(e) **Defaults** — `no-referrer` (most strict).

(f) **API contract** — `HelmetOptions.referrerPolicy: boolean | Token |
Token[]`. `Helmet.referrerPolicy(input?)`.

### F6 · Browser Feature Permissions

(a) **Scope** — `Permissions-Policy`, `Permissions-Policy-Report-Only`.

(b) **Conformance** — W3C Permissions Policy + W3C registry
`features.md` (https://github.com/w3c/webappsec-permissions-policy/blob/main/features.md).
RFC 9651 sf-dictionary serialisation.

(c) **Security guarantees**
- All Standardized features default-deny (`feature=()`) when policy
  enabled, except `sync-xhr=(self)` (OWASP backwards-compat).
- `ch-ua-*` Client Hints excluded from default-deny (separate opt-in
  lifecycle).
- Origin allowlist entries emitted as RFC 9651 sf-string (quoted) per
  spec; `'self'` and `*` emitted as bare sf-tokens.

(d) **Validation strictness**
- Feature-name grammar: `[a-z][a-z0-9-]{0,63}`.
- Origin: must parse via `URL`, `origin` non-`null`, scheme http/https.
- Unknown features (not in our internal `KNOWN_FEATURES`) emit a
  `HelmetWarningReason.UnknownPermissionsPolicyFeature` warning, do not
  fail.
- Per-feature allowlist cap `LIMITS.permissionsPolicyAllowlist` (32);
  total feature cap `LIMITS.permissionsPolicyFeatures` (64).

(e) **Defaults** — `permissionsPolicy: true` → all Standardized features
denied.

(f) **API contract** — `HelmetOptions.permissionsPolicy` /
`permissionsPolicyReportOnly`. `Helmet.permissionsPolicy(input?)`.

---

### F7 · Cross-Origin Isolation

(a) **Scope** — `Cross-Origin-Embedder-Policy` (+ Report-Only),
`Cross-Origin-Opener-Policy` (+ Report-Only),
`Cross-Origin-Resource-Policy`, `Origin-Agent-Cluster`,
`Document-Isolation-Policy` (+ Report-Only).

(b) **Conformance** — WHATWG HTML §7.1.3.1 (COOP), §7.1.4.1 (COEP),
WHATWG Fetch (CORP), HTML Origin §7.7 (OAC), WICG Document Isolation
draft.

(c) **Security guarantees**
- COEP `credentialless` value supported.
- COOP `noopener-allow-popups` (newer) supported.
- COOP/COEP `report-to` parameter cross-validated against
  `Reporting-Endpoints` — undeclared endpoint names throw.
- OAC emitted as RFC 9651 sf-boolean (`?1` / `?0`).

(d) **Validation strictness** — exhaustive enums per spec; each invalid
value yields a dedicated reason (`InvalidCorpValue`,
`InvalidCoopCoepValue`, `InvalidDocumentIsolationPolicyValue`).

(e) **Defaults** — COOP=`same-origin`, CORP=`same-origin`, OAC=`?1`.
COEP off by default (it can break embeds; opt-in).

(f) **API contract** — boolean | enum-token | `{ value, reportTo }`
object form for COOP/COEP. CORP token only. OAC boolean.

### F8 · Violation Reporting

(a) **Scope** — `Reporting-Endpoints`, `Report-To` (synthesised for NEL
backwards compat), `NEL`, plus `parseCspReport` ingestion utility.

(b) **Conformance** — W3C Reporting-1 (https://www.w3.org/TR/reporting-1/),
W3C Network Error Logging.

(c) **Security guarantees**
- Endpoint URLs must be HTTPS (`reporting_endpoint_not_https`).
- URL fragment stripped before serialisation (W3C Reporting-1 §3.6).
- Endpoint **name** restricted to RFC 9651 sf-dict-key grammar
  (`[a-z][a-z0-9_.\-*]{0,63}` — lowercase only).
- `parseCspReport` reads at most `LIMITS.cspReportBodyBytes` (64 KiB),
  capped at `LIMITS.cspReportItems` (100), with 10-second timeout
  (`LIMITS.cspReportTimeoutMs`).
- Both `application/csp-report` (legacy) and `application/reports+json`
  (Reporting API) accepted.
- `Object.create(null)` used inside the parser to prevent prototype
  pollution from JSON input.

(d) **Validation strictness**
- NEL fields: `report_to` references known endpoint, `max_age` ≥ 0,
  fractions ∈ [0, 1].

(e) **Defaults** — disabled (must opt-in by configuring
`reportingEndpoints`).

(f) **API contract** — `HelmetOptions.reportingEndpoints`, `nel`,
`Helmet.endpoints` static helper. `parseCspReport(request)` exported.

### F9 · Session / Cache Control

(a) **Scope** — `Cache-Control`, `Pragma`, `Expires`, `Clear-Site-Data`.

(b) **Conformance** — RFC 9111 (HTTP Caching), RFC 8674
(Clear-Site-Data).

(c) **Security guarantees**
- `Cache-Control` value validated at create-time for CRLF / control
  chars / oversize so header injection cannot occur via
  `cacheControl.value`.
- Clear-Site-Data tokens emitted as RFC 9651 sf-string (double-quoted).

(d) **Validation strictness**
- Standard Clear-Site-Data tokens: `cache`, `cookies`, `storage`,
  `executionContexts`, `clientHints`, `*`. Chromium extras
  (`prefetchCache`, `prerenderCache`) emit a warning, not an error.
- Cache-Control value: non-empty, ≤ `LIMITS.headerValueBytes`, no C0
  controls / DEL.

(e) **Defaults** — Cache-Control: `no-store, max-age=0` when enabled.
Clear-Site-Data: opt-in.

(f) **API contract** — `HelmetOptions.cacheControl`,
`HelmetOptions.clearSiteData`.

---

### F10 · Resource Integrity & Document Policy

(a) **Scope** — `Integrity-Policy` (+ Report-Only), `Document-Policy`
(+ Report-Only), `Require-Document-Policy`.

(b) **Conformance** — W3C Subresource Integrity (Integrity-Policy
draft), WICG Document Policy draft.

(c) **Security guarantees**
- Integrity-Policy emitted as RFC 9651 sf-dictionary
  (`blocked-destinations`, `sources`, `endpoints`).
- Document-Policy supports per-key parameters (`feature=value;param=v`)
  per WICG draft; parameter keys must satisfy RFC 9651 §3.1.2
  `lcalpha [...]`.

(d) **Validation strictness**
- Integrity-Policy: `blocked-destinations` ⊆ {`script`, `style`};
  `sources` ⊆ {`inline`}; `endpoints` cross-referenced with
  `reportingEndpoints`.
- Document-Policy entries: cap `LIMITS.documentPolicyEntries` (64).

(e) **Defaults** — disabled.

(f) **API contract** — `HelmetOptions.integrityPolicy`,
`integrityPolicyReportOnly`, `documentPolicy`,
`documentPolicyReportOnly`, `requireDocumentPolicy`.

### F11 · Information Disclosure Minimisation

(a) **Scope** — `removeHeaders` (server identification strip),
`X-Permitted-Cross-Domain-Policies`, `X-Download-Options`,
`X-XSS-Protection`, `X-Robots-Tag`, `Timing-Allow-Origin`,
`X-DNS-Prefetch-Control`.

(b) **Conformance** — OWASP Secure Headers; Adobe Cross-Domain Policy;
W3C Resource Timing §3.5.2; Google Search Central (X-Robots-Tag).

(c) **Security guarantees**
- Default-strip set: `server`, `x-powered-by`, `x-aspnet-version`,
  `x-aspnetmvc-version` (OWASP must-strip).
- Optional `removeHeaders: 'owasp'` strips the full OWASP list (70
  headers) per `headers_remove.json` 2026-03-05.
- X-XSS-Protection restricted to `0` and `1; mode=block` (modern
  recommendation `0`).
- `Timing-Allow-Origin` restricted to `*`, `null`, or the ASCII
  serialisation of an origin (no path/query/fragment) per Resource
  Timing §3.5.2.
- `X-Robots-Tag` directives validated against Google Search Central
  spec (boolean rules + value-rules + optional bot-name prefix +
  CRLF/comma rejection).

(d) **Validation strictness**
- `X-Permitted-Cross-Domain-Policies` ∈ {`none`, `master-only`,
  `by-content-type`, `by-ftp-filename`, `all`}.
- `X-DNS-Prefetch-Control` ∈ {`on`, `off`} (case-sensitive).

(e) **Defaults** — XPCDP=`none`, X-DNS-Prefetch=`off`,
X-XSS-Protection unset, X-Download-Options unset, X-Robots-Tag unset,
Timing-Allow-Origin unset, removeHeaders=`true` (must-strip set).

(f) **API contract** — see HelmetOptions of same names.

### F12 · Cryptographic Builders

(a) **Scope** — `Helmet.generateNonce`, `hashFromString`, `lintCsp`,
`Csp` keyword/nonce/hash builders.

(b) **Conformance** — CSP3 §2.3.1 (nonce / hash sources), W3C SRI
(Subresource Integrity), Google csp-evaluator heuristics.

(c) **Security guarantees**
- `Helmet.generateNonce(bytes?)` uses `crypto.getRandomValues` (CSPRNG).
- Default 16 bytes (128-bit entropy floor per OWASP).
- `bytes` argument MUST yield ≥ 16 base64url chars (≥ 12 random bytes
  → ≥ 16 chars after base64url) so the result is always usable as a
  CSP nonce-source per CSP3 §2.3.1.
- `hashFromString` uses Web Crypto SubtleCrypto.digest (no custom hash
  implementations).
- `Csp.nonce(value)` and `Csp.hash(algo, value)` are type-only sugar;
  validation happens when the source list is processed.

(d) **Validation strictness**
- `lintCsp` reports findings with severity `high` / `medium` /
  `low`; behaviour is advisory, never throws.
- Nonce input to `headers({ nonce })` validated against the same
  base64url charset and length floor as `Helmet.generateNonce` produces.

(e) **Defaults** — N/A (utilities).

(f) **API contract** — `Helmet.generateNonce(bytes?: number): Nonce`,
`hashFromString(input, algo?): Promise<string>`,
`lintCsp(directives, opts?): readonly CspLintFinding[]`,
`Csp` re-export.

## 3. Browser/runtime support matrix

| Runtime | Status |
|---|---|
| Bun ≥ 1.0 | Primary target — `Bun.serve(req → helmet.apply(res))` |
| Node ≥ 18 | Supported (Web Fetch API native) |
| Deno ≥ 1.30 | Supported |
| Cloudflare Workers | Supported |
| Vercel Edge / Next.js Edge | Supported |
| Hono / Fastify-fetch | Supported (Web Fetch API) |
| Express (legacy `res.setHeader`) | Adapter required (`headersRecord()` + `res.set(record)`) |
| Node ≤ 16 | Not supported (no native Headers API) |

Browser-side: the package never runs in browsers; it only emits headers
the browser parses. Browser conformance to the cited specs is the
browser vendor's responsibility.

## 4. Known scope-out items (intentional)

These are NOT defects in this package and will not be added:

- HPKP (`Public-Key-Pins`) — deprecated, removed from browsers
- `Expect-CT` — deprecated (CT now mandatory in browsers)
- `Feature-Policy` — superseded by `Permissions-Policy`
- `X-XSS-Protection: 1` (legacy enable) — known unsafe; only `0` and
  `1; mode=block` accepted
- Express middleware (`(req, res, next) =>`) — use the adapter pattern
  with `headersRecord()`
- CORS / preflight — `@zipbul/cors`
- Rate limiting — `@zipbul/rate-limiter`

## 5. Defect classification rules (reference)

Given the above clauses, a finding is a:

| Class | Trigger |
|---|---|
| **must-fix** | Violates any §1 cross-cutting rule, any §2 (b)/(c)/(d) clause, or §F12 entropy floor |
| **defect-but-deferred** | Spec is in flux (WICG/W3C draft) and our implementation matches the latest published draft |
| **out of scope** | Not stated in §1 / §2 |
| **non-defect** | Cosmetic (comments, naming preference, refactor opportunity without behavioural impact) |

## 6. Versioning

This SPEC.md is updated whenever a §1 or §2 clause changes. SemVer:
- Adding a feature → minor
- Tightening validation that was previously too permissive → minor (no
  user code becomes invalid that *was* spec-valid before)
- Loosening validation that was rejecting spec-valid input → patch
  (bug fix)
- Breaking change to `HelmetOptions` shape, `HelmetError.violations[].reason`
  enum value, or compiled header value semantics → major.
